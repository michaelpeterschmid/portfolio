Connect 4 3D



Verwendete Technologien:

  • HTML
  • Pico CSS
  • JavaScript
  • Socket.io

Beschreibung

Dies war ein Hobby-Projekt von mir. „4 Gewinnt 3D“ habe ich zum ersten Mal in klassischer Form auf einer Holzspielplatte mit meinem Cousin gespielt. Schon bald wurde es zu einem meiner Lieblings-Strategiespiele, da es sehr einfach zu verstehen ist und dennoch eine erstaunliche Spieltiefe bietet. Da ich keine digitale Version des Spiels fand, kam mir die Idee, eine Web-App zu entwickeln, um es online mit meinen Freunden spielen zu können. Die Entwicklung der digitalen Version ermöglichte es mir, sowohl an der Spiel­logik als auch an der Multiplayer-Implementierung zu tüfteln. Die erste Version entstand während meiner Lehre (ca. 2021), kürzlich habe ich das Projekt wieder aufgenommen, neue Features hinzugefügt und es im Google Play Store veröffentlicht. Das Backend für die Multiplayer Komponente wird auf Render gehostet.

Arbeitszeiterfassung



Verwendete Technologien:

  • Express.js
  • Knex.js
  • MySQL
  • Azure Active Directory
  • Insomnia REST

Beschreibung

Dies war ein Projekt im Rahmen meiner IPA für die Technische Fachschule Bern. Ziel war es, ein Backend-Modul für eine Arbeitszeiterfassung zu entwickeln. Umgesetzt wurde es als JavaScript-Webapplikation mit Node.js und Express.js, einer MySQL-Datenbank und Knex.js zur Datenbankabstraktion, die Anmeldung erfolgt über Azure Active Directory. Es wurde eine REST-Schnittstelle (API) definiert, über die Lernende ihre Arbeitszeiten erfassen, ändern, löschen und abrufen können. Der Nutzen liegt in einer modularen, erweiterbaren Lösung, die präzise Arbeitszeitdaten für Lernende und Berufsbildner bereitstellt und später in bestehende Architekturen integriert werden kann. Zusammengefasst ermöglicht die Anwendung einfache Zeiterfassung, Auswertung und Verwaltung für interne und kundenbezogene Projekte. Die Technische Fachschule Bern hat diese Lösung nach dem Abschluss meiner IPA eingeführt und verwendet.

Gomoku



Verwendete Technologien:

  • HTML
  • CSS
  • JavaScript
  • Socket.io

Beschreibung

Gomoku, auch bekannt als "Fünf in einer Reihe", ist ein traditionelles Strategiespiel, das seinen Ursprung in Ostasien hat. Es wird auf einem Gitterbrett gespielt, wobei zwei Spieler abwechselnd Steine platzieren, um eine ununterbrochene Reihe von fünf Steinen horizontal, vertikal oder diagonal zu bilden. In meiner digitalen Version von Gomoku habe ich sowohl die Spielmechanik als auch die Multiplayer-Funktionalität implementiert. Das Spiel ermöglicht es den Spielern, online gegeneinander anzutreten, was durch die Verwendung von Socket.io für die Echtzeit-Kommunikation zwischen den Spielern erreicht wird. Die Benutzeroberfläche wurde mit HTML und CSS gestaltet, um ein ansprechendes und benutzerfreundliches Erlebnis zu bieten. JavaScript wurde verwendet, um die Spiellogik zu implementieren und die Interaktivität zu gewährleisten.
